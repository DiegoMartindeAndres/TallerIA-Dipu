## 💡 Preguntas clave

| Pregunta | Respuesta |
|----------|-----------|
| ¿Qué es un prompt del sistema? | Las instrucciones base que definen cómo actúa la Gema |
| ¿Cuándo se escribe? | Una sola vez, al crear la Gema |
| ¿Cuáles son los componentes? | ROL, OBJETIVO, CONTEXTO, PROCEDIMIENTO, RESTRICCIONES |
| ¿Cuánto tiempo toma escribirlo? | 1-2 horas la primera vez |
| ¿Puedo mejorar el mío después? | Claro, siempre puedes optimizar |

- [ ] Añadí **RESTRICCIONES** claras

## 🔍 Antes de empezar

Tendrás que decidir:

1. **¿Qué problema va a resolver mi Gema?**
   - Ejemplo: "Analizar si solicitudes cumplen requisitos"

2. **¿A quién va a servir?**
   - Ejemplo: "Al equipo de Personal"

3. **¿Qué normativa aplica?**
   - Ejemplo: "LPAC, LOFAGE, convenio local"

## ❓ Preguntas frecuentes

**P: ¿Mi prompt tiene que ser perfecto?**  
R: No. Empieza con algo funcional y mejóralo iterativamente.

**P: ¿Cuánto de largo debe ser?**  
R: Entre 300-800 palabras es lo típico. Más detalle = mejores respuestas.

**P: ¿Puedo copiar prompts de otros?**  
R: Claro, úsalos como base y adapta a tu contexto específico.

**P: ¿Qué pasa si falta algo?**  
R: Las respuestas serán vagas o inconsistentes. Revisa los 5 componentes.
